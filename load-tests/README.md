# Load Tests — Doctor Prescribing Flow

## Prerequisites

1. **Install k6**
   ```
   winget install k6
   ```
   Or download from: https://k6.io/docs/getting-started/installation/

2. **Node dependencies installed** in the `backend/` directory

## Files

| File | Purpose |
|------|---------|
| `prescribing-flow.js` | k6 test script for Doctor prescribing flow |
| `seed-load-test.ts` | Seeds 10 doctors × 100 patients each |
| `test-doctors.json` | Generated by seed script — doctor credentials |
| `run.cmd` | One-click runner (Windows) |
| `results.csv` | Output — test results |

## How to Run

### Option 1: One-click (Windows)
```
load-tests\run.cmd
```

### Option 2: Manual steps

1. **Seed test data**
   ```
   cd backend
   set NODE_ENV=test
   npx ts-node prisma/seed-load-test.ts
   ```

2. **Start backend in test mode**
   ```
   cd backend
   set NODE_ENV=test
   npx ts-node src/server.ts
   ```

3. **Run k6 test**
   ```
   cd load-tests
   k6 run prescribing-flow.js
   ```

## Test Scenarios

| Phase | Duration | VUs | Load |
|-------|----------|-----|------|
| Warm-up | 30s | 1 → 5 | Gradual ramp |
| Steady | 2 min | 5 | Normal load |
| Spike | 30s | 5 → 20 | Peak load |
| Peak | 1 min | 20 | High load |
| Cooldown | 30s | 20 → 0 | Drain |

## What Gets Tested

Each virtual doctor (VU) repeats this flow:
1. `POST /api/auth/login` — authenticate
2. `GET /api/patients` — pick a random patient
3. `POST /api/prescriptions` — create prescription with medicines + investigations
4. `GET /api/prescriptions` — list prescriptions (pagination)
5. `GET /api/prescriptions/:id/pdf` — download PDF (only VUs 1-10 to reduce load)

## Thresholds (pass/fail criteria)

- **95% of requests** under 2s
- **Average** under 800ms
- **Error rate** < 2%
- **Prescription creation success** > 95%
- Login p95 < 1s, List patients p95 < 1s, PDF download p95 < 5s

## Customizing

- Change target URL: `set BASE_URL=http://your-server:5000/api`
- Adjust load profile: edit `stages` in `prescribing-flow.js`
- Change test data volume: edit `DOCTOR_COUNT` / `PATIENTS_PER_DOCTOR` in `seed-load-test.ts`
